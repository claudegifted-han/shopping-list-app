## 2. 일일 운영

### 2.1 시스템 상태 확인

**매일 아침 (09:00)**

```bash
# 1. Claude Code 정상 작동 확인
claude --version

# 2. MCP 서버 연결 확인

@교무운영부 시스템 상태 확인

# 3. Notion 연결 확인
# Notion 데이터베이스 접근 테스트
```

## 5. 장애 대응

### 5.1 장애 등급

| 등급 | 정의 | 대응 시간 |
|------|------|----------|
| 긴급 (P1) | 전체 시스템 중단 | 1시간 이내 |
| 높음 (P2) | 주요 기능 장애 | 4시간 이내 |
| 보통 (P3) | 일부 기능 이상 | 1일 이내 |
| 낮음 (P4) | 경미한 불편 | 1주일 이내 |

### 5.2 장애 대응 절차

```
1. 장애 인지
   ↓
2. 영향 범위 파악
   ↓
3. 긴급 조치 (필요시)
   ↓
4. 원인 분석
   ↓
5. 해결 조치
   ↓
6. 정상화 확인
   ↓
7. 사후 보고
```

### 5.3 주요 장애 유형 및 대응

#### Claude Code 연결 실패

```
증상: Claude Code 명령 실행 안됨

확인:
1. 인터넷 연결 상태
2. Claude Code 버전 확인
3. API 키 유효성

조치:
1. 네트워크 재연결
2. Claude Code 재시작
3. API 키 재설정
```

#### Notion MCP 연결 실패

```
증상: Notion 데이터 조회/수정 안됨

확인:
1. NOTION_API_KEY 환경변수
2. MCP 서버 상태
3. Notion Integration 연결

조치:
1. 환경변수 재설정
2. MCP 서버 재시작
3. Notion Integration 재연결
```

#### 에이전트 응답 오류

```
증상: 잘못된 응답, 형식 오류

확인:
1. 요청 내용 명확성
2. 프롬프트 버전
3. 템플릿 파일 상태

조치:
1. 요청 재구성
2. 프롬프트 확인/수정
3. 템플릿 복구
```
